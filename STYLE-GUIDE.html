Section 01
Design Philosophy
This is a diegetic interface — it presents itself as a terminal that exists inside the Star Wars galaxy, not a website that sits on top of it. Every panel reads like a readout on a ship's navigation computer or a Rebel briefing console. The UI is never decorative: borders encode state, color encodes data type, motion encodes signal reception. The galaxy grid is the primary object; all other panels are instruments that serve it.
Core Tension The app contains ~5,900 systems — dense data. The aesthetic must make density legible, not overwhelming. Restraint in decoration is what lets the data breathe.
Canon vs. User Two classes of data exist: seeded canon systems and user-added entries. The visual language must distinguish them without hierarchy — canon is not "better," just differently sourced.

Section 04 — Primary View
Galaxy Grid (20×21)

Sparse — 1–3 systems (faint cyan)
Dense — 4–12 systems (mid cyan)
Core — 13+ systems (bright cyan)
Selected — yellow bracket corners + yellow dot
User-added system present — dashed yellow
Restricted / notable — faint red
Column & Row Labels
Axis labels (C–V, 1–21) sit outside the grid in a fixed header row and left column. They use 8px uppercase gray text. On hover of any cell, the corresponding column letter and row number highlight in green — a crosshair effect that helps the user locate themselves on the map without cluttering the cells. Selected coordinate labels turn yellow.
Density Encoding
Cell fill is a 4-stop opacity ramp on Hologram Cyan — the only place in the UI where a continuous value is expressed through color intensity. This is intentional: the galaxy map IS a density heatmap. The ramp goes: empty (#090909) → sparse (6% cyan) → dense (16% cyan) → core (28% cyan). No further stops — keep it readable at 20×21 scale.

Error / Alert

All corners are 90° sharp — no border-radius anywhere in the UI. The Star Wars aesthetic is rectilinear: the only curves are in SVG icons (radar circles, target reticles) and are always intentional geometry, never cosmetic softening.
Section 09
Motion & Interaction Behavior
Grid Cell Click
Panel opens instantly (0ms delay). The selected cell's border transitions from gray → yellow in 80ms. No animation on the panel itself — it appears.
Grid Cell Hover
Column letter + row number in the axis labels turn green. The hovered cell gets a 1px green border flash (100ms). This is the only use of scanner green in the interactive grid.
Data Loading
Loading state: a slow horizontal scan line sweeps across the panel in green (3s linear, repeating). Not a spinner — a sweep. Text reads "SCANNING…" in 8px gray uppercase. Never a skeleton screen.
System Row Hover
Background shifts from transparent → --surface2 (#141414) in 100ms. No scale, no shadow, no underline. Border color on the row does not change — only background fills.
Error / Validation
Error bar appears below form with red border + red text. Input border flashes red for 300ms then holds. No shake animation — errors are silent and stationary in this universe.
Page Transitions
Grid → detail navigation is an instant replace (React Router). No slide, no fade. The destination page's content appears as if received by signal — coordinates update in the header immediately. Continuity comes from the persistent grid, not from transitions.
Section 10
Design Constraints
Permitted
✓  Near-black backgrounds only
✓  Monospace font throughout
✓  Uppercase text everywhere
✓  Sharp 90° corners only
✓  1px or sub-pixel borders
✓  Cyan opacity ramp for density
✓  Dashed yellow for user data
✓  Italic gray for null fields
✓  Aurebesh in static decorative labels
✓  Asymmetric panel layouts
✓  Color used only for data meaning
✓  Instant state transitions (≤100ms)
Prohibited
✗  Rounded corners or border-radius
✗  Drop shadows, glows, blur
✗  Gradients (except grid density ramp)
✗  Light / white backgrounds
✗  Proportional or sans-serif fonts
✗  Smooth counter / number animations
✗  Skeleton screens or shimmer loaders
✗  Tooltips — use panel context instead
✗  Mixed-case text in UI labels
✗  Icons from general icon libraries
✗  Bright fills on panel backgrounds
✗  Editing controls on canon systems
